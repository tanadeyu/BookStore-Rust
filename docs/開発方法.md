## 基本のコマンド

### プロジェクトのクローン

```bash
git clone https://github.com/tanadeyu/BookStore-Rust.git
cd BookStore-Rust
```

### 依存関係の取得とビルド

```bash
# 初回のみ
cargo build

# 実行
cargo run
```

### よく使う Cargo コマンド

| コマンド | 説明 |
|---------|------|
| `cargo build` | デバッグビルド |
| `cargo build --release` | リリースビルド（最適化付き） |
| `cargo run` | ビルド＆実行 |
| `cargo check` | 高速なコンパイルチェック（ビルドなし） |
| `cargo test` | テスト実行 |
| `cargo clean` | ビルドキャッシュ削除 |
| `cargo doc` | ドキュメント生成 |

## ホットリロード開発

ファイル変更時に自動的に再ビルドしてくれるツールです。

### cargo-watch のインストール

```bash
cargo install cargo-watch
```

### 使用方法

```bash
# ファイル変更時に自動で `cargo run` を実行
cargo watch -x run

# ファイル変更時に自動で `cargo check` を実行
cargo watch -x check

# カスタムコマンド
cargo watch -x 'run --bin BookStore-Rust'
```

**メリット:**
- ファイル保存するたびに自動リビルド
- 手動で `cargo run` を叩く必要がない
- 開発効率が大幅に向上

## デバッグ方法

### 1. println! デバッグ

最もシンプルな方法です。

```rust
async fn get_products(pool: web::Data<SqlitePool>) -> Result<HttpResponse, actix_web::Error> {
    println!("🔍 get_products called");

    let products = sqlx::query_as::<_, models::Product>(/* ... */)
        .fetch_all(pool.get_ref())
        .await?;

    println!("📊 Found {} products", products.len());

    Ok(HttpResponse::Ok().json(products))
}
```

### 2. VS Code / Trae でデバッグ実行

**`.vscode/launch.json` の作成:**

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "type": "lldb",
            "request": "launch",
            "name": "Debug executable 'BookStore-Rust'",
            "cargo": {
                "args": [
                    "build",
                    "--bin=BookStore-Rust"
                ],
                "filter": {
                    "name": "BookStore-Rust",
                    "kind": "bin"
                }
            },
            "args": [],
            "cwd": "${workspaceFolder}"
        }
    ]
}
```

**使用方法:**
1. ソースコードの左側をクリックしてブレークポイントを設定
2. `F5` でデバッグ開始
3. ブレークポイントで停止し、変数の値を確認

### 3. ログ出力

環境変数でログレベルを設定：

```bash
# Windows (PowerShell)
$env:RUST_LOG="debug"

# Windows (Git Bash)
export RUST_LOG=debug

# 実行
cargo run
```

## コードの品質チェック

### Clippy（リンター）

Rustのコーディング規約に違反していないかチェックします。

```bash
# インストール
rustup component add clippy

# 実行
cargo clippy

// 自動修正可能な警告を自動修正
cargo clippy --fix
```

### rustfmt（フォーマッター）

コードのスタイルを統一します。

```bash
# インストール
rustup component add rustfmt

# 実行
cargo fmt

# チェックのみ（変更はしない）
cargo fmt --check
```

## トラブルシューティング

### コンパイルエラー

**エラー例:**
```
error[E0425]: cannot find value `pool` in this scope
```

**対処法:**
1. エラーメッセージをよく読む
2. 該当する変数がスコープ内にあるか確認
3. `cargo clean` でキャッシュ削除後に再ビルド

### 依存関係の問題

**エラー例:**
```
error: failed to compile`
```

**対処法:**
```bash
# Cargo.lock を削除して再ビルド
rm Cargo.lock
cargo build
```

### データベースエラー

**エラー例:**
```
Error: Database(SqliteError { code: 14, message: "unable to open database file" })
```

**対処法:**
```bash
# データベースファイルを削除して再作成
rm bookstore.db
cargo run
```

### ポートが使用中

**エラー例:**
```
Error: Os { code: 10048, kind: AddrInUse, ... }
```

**対処法:**
```bash
# ポートを使用しているプロセスを確認
netstat -ano | grep 8080

# プロセスを終了（Windows）
taskkill //F //PID <プロセスID>
```

## 開発ワークフロー

1. **機能追加の流れ:**
   ```bash
   cargo watch -x run   # ターミナル1: ホットリロード監視
   cargo test           # ターミナル2: テスト実行
   ```

2. **コミット前のチェック:**
   ```bash
   cargo fmt            # フォーマット
   cargo clippy         # リンター
   cargo test           # テスト
   ```

3. **リリースビルド:**
   ```bash
   cargo build --release
   ./target/release/BookStore-Rust.exe
   ```
